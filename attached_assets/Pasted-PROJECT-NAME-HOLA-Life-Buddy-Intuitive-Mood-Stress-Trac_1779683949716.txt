PROJECT NAME
HOLA! Life Buddy — Intuitive Mood & Stress Tracker

PROJECT OVERVIEW
Build a full-stack mental wellness platform called "HOLA! Life Buddy" with two separate apps that are connected through a shared backend:

1. ADMIN/PSYCHOLOGIST APP — Web dashboard (desktop-first) + companion mobile app
2. USER MOBILE APP — Mobile-only app for patients/users

Both apps share the same backend API and database. They communicate in real-time so psychologists can monitor their clients' wellness data, assign exercises, and receive alerts — while users enjoy a private daily wellness companion.

DESIGN SYSTEM
UI Style: Minimalist — inspired by Threads (Meta) and Quabble
Color Palette:
  - Background: #FFFFFF (light) / #0A0A0A (dark)
  - Surface: #F3F3F3 (light) / #181818 (dark)
  - Border: #E0E0E0 (light) / #2A2A2A (dark)
  - Primary accent: #000000 (light) / #FFFFFF (dark)
  - Secondary accent: #6B6B6B
  - Positive/calm: #3DD68C (soft green)
  - Alert/stress: #FF6B6B (soft red)
  - Warning: #FFB547 (amber)
  - Info: #5B9CF6 (soft blue)

Typography:
  - Font: Inter (Google Fonts)
  - Weights: 400 (body), 500 (labels), 600 (headings only)
  - Base size: 14px mobile, 15px web
  - Line height: 1.6

Design Rules:
  - No gradients, no heavy shadows — use 1px borders and subtle background fills
  - Generous whitespace, bottom tab navigation on mobile
  - Rounded corners: 12px cards, 8px buttons, 999px pills/badges
  - Icons: use Lucide React icon set throughout
  - Animations: subtle fade and slide transitions only (no bouncing)
  - Dark mode support from day one (use CSS variables or Tailwind dark:)

TECH STACK
Frontend (Web Admin): React + Vite + Tailwind CSS
Frontend (Mobile — both apps): React Native + Expo
Backend: Node.js + Express.js
Database: PostgreSQL with Prisma ORM
Auth: JWT-based with role system (admin / psychologist / user)
Real-time: Socket.io (for alerts and messaging)
AI Chatbot: Anthropic Claude API (claude-sonnet-4-20250514)
Push Notifications: Expo Push Notifications
File/Audio Storage: Cloudinary or local /public folder for MVP
State Management: Zustand (lightweight, no Redux)
API: RESTful JSON API with versioning (/api/v1/)

DATABASE SCHEMA (KEY TABLES)
Users: id, name, email, password_hash, role (psychologist | user), avatar, created_at
Psychologist_Profile: id, user_id, license_no, specialization, bio
Client_Assignments: id, psychologist_id, user_id, status, linked_at
Mood_Entries: id, user_id, mood_score (1–10), emotions[], note, created_at
Journal_Entries: id, user_id, title, content, prompt_used, is_shared_with_psychologist, created_at
Meditation_Sessions: id, user_id, module_name, duration_seconds, completed_at
Chat_Messages: id, user_id, role (user | assistant), content, created_at
Assigned_Tasks: id, psychologist_id, user_id, task_type, task_data (JSON), due_date, completed_at
Alerts: id, psychologist_id, user_id, type, message, is_read, triggered_at
Appointments: id, psychologist_id, user_id, scheduled_at, notes, status

AI CHATBOT SYSTEM PROMPT
Use this as the Claude API system prompt for HOLA Buddy:

"You are HOLA Buddy, a warm, non-judgmental AI wellness companion inside the HOLA! Life Buddy app. You support users with evidence-based techniques drawn from CBT (Cognitive Behavioral Therapy) and DBT (Dialectical Behavior Therapy). Your tone is friendly, calm, and encouraging — like a caring friend who also knows psychology. Keep responses concise (2–4 sentences max unless guiding an exercise). Never diagnose. Always encourage professional help for serious concerns. When a user seems in crisis, gently direct them to their psychologist or a helpline. You can guide breathing exercises, suggest journaling prompts, help reframe negative thoughts, and celebrate small wins. Always validate feelings before offering techniques."

ALERT LOGIC (BACKEND)
Auto-trigger an alert to the linked psychologist when:
- User mood score is 3 or below for 3 consecutive days
- User has not logged in for 5+ days
- Journal entry contains keywords: (hopeless, worthless, can't go on, end it, give up) — flag for review only, do NOT read content automatically
- Assigned task is 3+ days overdue

9. Add secure messaging (Socket.io)
10. Add Mindfulness tools (breathing animation, journal, CBT tools)
11. Add Progress screen and Community feed
12. Polish: dark mode, animations, push notifications

IMPORTANT NOTES FOR REPLIT
- Use .env for all secrets: DATABASE_URL, JWT_SECRET, ANTHROPIC_API_KEY, CLOUDINARY_URL
- For MVP, mock the audio files with placeholder URLs — implement real audio later
- Keep the UI strictly minimalist: black, white, and one accent color per section
- All screens should support both light and dark mode
- Start with the backend and mobile user app — the admin web dashboard can come after
- Use Expo Go for mobile testing during development
- Comment all major functions clearly for easy iteration
